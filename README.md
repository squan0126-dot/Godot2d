# 📖 小说创作工作台 (Novel Studio)

一款**纯本地运行**的小说创作管理工具，无需联网、无需注册，数据全部保存在本地 JSON 文件中。支持人物管理、关系图谱、地点管理、情节板、钩子/伏笔追踪、时间线、章节大纲、正文编辑等全流程创作功能。

---

## ✨ 功能特性

| 模块 | 功能说明 |
|------|----------|
| 🎯 仪表盘 | 项目概览、创作进度统计、快速导航 |
| 👥 人物管理 | 创建角色卡片（姓名、身份、描述、标签、备注、颜色标记） |
| 🔗 关系图谱 | 可视化人物关系网络，支持拖拽布局 |
| 📍 地点管理 | 场景/地点信息管理（类型、描述、备注） |
| 📋 情节板 | 看板式情节管理，按幕/阶段分列展示 |
| 🪝 钩子/伏笔 | 追踪埋设的钩子和伏笔（埋设点、揭示点、状态） |
| 🔄 反转设计 | 管理故事反转（前置铺垫、反转点、后续影响） |
| 📅 时间线 | 按时间顺序排列事件，标注重要程度 |
| 📝 章节管理 | 章节大纲、字数目标、状态追踪 |
| ✍️ 正文编辑 | 分章节正文写作，支持字数统计 |
| 📦 素材库 | 收集创作素材和灵感 |
| 📊 数据分析 | 创作数据统计与分析 |

## 🚀 快速开始

### 环境要求

- **Python 3.x**（任意版本，用于启动本地服务器）
- **浏览器**（Chrome / Edge / Firefox 等现代浏览器）

> 💡 不需要安装任何第三方库，Python 标准库即可运行。

### 安装步骤

#### 方式一：双击启动（推荐）

1. 下载/克隆本项目到本地
2. 双击 `novel-workspace/启动工作台.bat`
3. 浏览器会自动打开 `http://localhost:8266`
4. 开始创作！

#### 方式二：命令行启动

```bash
# 进入项目根目录
cd novel-studio

# 启动服务器
python server.py
```

然后在浏览器中访问：**http://localhost:8266**

## 📖 使用教程

### 1. 创建项目

启动后，点击左侧「仪表盘」，在顶部可以看到项目名称。点击右上角设置可修改：
- 作品名称
- 作品简介
- 作者
- 类型/标签

### 2. 创建人物

1. 点击左侧「👥 人物」
2. 点击右上角「+ 新建人物」
3. 填写人物信息：
   - **姓名**：角色名
   - **身份**：主角/配角/反派等
   - **描述**：外貌、性格、背景
   - **标签**：用逗号分隔的关键词
   - **颜色**：用于关系图谱中的节点颜色
   - **备注**：创作笔记

### 3. 建立人物关系

1. 点击左侧「🔗 关系图谱」
2. 点击「+ 添加关系」
3. 选择两个人物，填写关系类型和描述
4. 图谱会自动生成可视化网络，支持拖拽调整位置

### 4. 管理地点

1. 点击左侧「📍 地点」
2. 点击「+ 新建地点」
3. 填写地点名称、类型（城市/村落/废墟等）、描述和备注

### 5. 规划情节

1. 点击左侧「📋 情节板」
2. 情节按列（幕）组织，可自定义列名
3. 点击「+ 新建情节」添加情节卡片
4. 每张卡片包含标题和描述

### 6. 埋设钩子/伏笔

1. 点击左侧「🪝 钩子伏笔」
2. 点击「+ 新建」
3. 填写：
   - **标题**：钩子名称
   - **类型**：钩子 / 伏笔
   - **状态**：open / resolved
   - **埋设点**：在哪里埋下
   - **揭示点**：在哪里揭开
   - **备注**：创作意图

### 7. 设计反转

1. 点击左侧「🔄 反转设计」
2. 记录每个反转的前置铺垫、反转点和后续影响

### 8. 编排时间线

1. 点击左侧「📅 时间线」
2. 按时间顺序添加事件
3. 标注重要程度（high/medium/low）
4. 关联相关人物

### 9. 章节管理与正文写作

1. 点击左侧「📝 章节」创建章节大纲
2. 每章包含：标题、状态、摘要、情绪、目标字数、涉及人物/地点、大纲
3. 点击「✍️ 正文编辑」进入写作模式
4. 选择章节后直接编写正文

### 10. 保存与同步

- **保存**：点击右上角「💾 保存」，数据写入本地 JSON 文件
- **同步**：点击右上角「🔄 同步」，从本地文件重新加载数据

> ⚠️ 数据保存在 `novel-workspace/data/` 目录下，建议定期备份或使用 Git 管理版本。

## 🔧 技术说明

### 架构

- **前端**：纯 HTML + CSS + JavaScript 单文件应用（无框架依赖）
- **后端**：Python 标准库 `http.server`（无需 pip install）
- **数据**：JSON 文件存储（人类可读，Git 友好）

### API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 返回工作台页面 |
| GET | `/api/data` | 获取所有数据 |
| POST | `/api/data` | 保存所有数据 |
| GET | `/api/save/{key}` | 获取单个数据文件 |
| POST | `/api/save/{key}` | 保存单个数据文件 |

### 端口配置

默认端口为 `8266`，如需修改，编辑 `server.py` 中的 `PORT` 变量。

## 💡 与 AI 协作

本工具的数据全部以 JSON 格式存储，天然适合与 AI 编程助手协作：

1. AI 可以直接读取 `data/` 目录下的 JSON 文件，了解你的故事全貌
2. AI 可以直接修改 JSON 文件来添加/编辑人物、情节等
3. 修改后在工作台点击「同步」即可看到最新内容
4. 正文写作时，AI 可以参考人物设定、情节大纲来保持一致性

## 🔄 数据备份与迁移

### 备份

整个 `novel-workspace/data/` 目录就是你的全部创作数据，复制即备份。

### 迁移到新设备

1. 将整个项目文件夹复制到新设备
2. 确保新设备安装了 Python 3.x
3. 双击 `启动工作台.bat` 即可继续创作

### 使用 Git 管理版本

```bash
git init
git add .
git commit -m "初始化小说项目"
```

## 📋 常见问题

**Q: 启动后浏览器打不开？**
A: 手动在浏览器地址栏输入 `http://localhost:8266`

**Q: 点击保存/同步没反应？**
A: 确认命令行窗口中的服务器还在运行（不要关闭黑色窗口）

**Q: 如何修改端口号？**
A: 编辑 `server.py`，修改 `PORT = 8266` 为你想要的端口

**Q: 数据存在哪里？**
A: `novel-workspace/data/` 目录下的 JSON 文件

**Q: 支持 Mac/Linux 吗？**
A: 支持！直接运行 `python server.py` 即可（.bat 脚本仅限 Windows）
