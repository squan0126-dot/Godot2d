# 2D场景交互设计规范

> **状态**：已确认方向  
> **核心决策**：不放小人进场景跑动，采用"大地图选地点 → 场景画面+热点交互"模式  
> **参考对标**：《逆转裁判》（场景热点）+ 《太吾绘卷》（地点选择）+ 《Darkest Dungeon》（氛围文字）  
> **设计原则**：每一次点击都有意义，不让玩家做无效操作

---

## 一、两种核心模式

### 1.1 模式定义

| 模式 | 触发条件 | 玩家操作 | 退出条件 |
|------|---------|---------|---------|
| **探索模式** | 进入任何地点 | 点击热点、选择地点、使用道具 | 触发关键事件 / 离开地点 |
| **叙事模式** | 触发关键事件 / 恐怖递进 / Boss登场 | 点击"继续"或做出选择 | 事件结束 / 进入战斗 |

### 1.2 模式切换规则

```
【探索模式】
  玩家自由点击热点 → 获取信息/触发对话/购买物品
       ↓
  点击某个热点触发"关键事件"
       ↓
【叙事模式】（画面渐变切入，不是弹窗）
  全屏背景 + 文字旁白 + 立绘对话
  恐怖递进 / 真相揭露 / Boss登场
       ↓
  选择分支（如有）
       ↓
【战斗模式】（如需要）
  符箓连携 → 绝技释放 → 击败敌人
       ↓
【处置选择】（驱赶/超度/封印/谈判）
       ↓
【回到探索模式】
```

**关键**：两种模式的切换必须**无缝**——整个画面渐变过渡，不是"弹出对话框"。

## 二、探索模式详细设计

### 2.1 界面结构

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│              【场景背景CG】（全屏）                       │
│                                                         │
│         ● 热点A        ● 热点B        ● 热点C          │
│                                                         │
│                    ● 热点D（隐藏，通幽后显示）            │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [地点名称]                          [通幽] [背包] [地图] │
│  [热点列表/文字提示区]                                    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 热点类型

| 热点类型 | 图标/表现 | 点击后效果 | 示例 |
|----------|----------|-----------|------|
| **NPC对话** | 人形图标/立绘缩影 | 进入对话（立绘+对话框） | 旅店老板、道士、商人 |
| **物品/线索** | 发光点/物品图标 | 获得物品或线索文字 | 告示栏、地上的符纸、墙上的血迹 |
| **场景入口** | 门/路径图标 | 切换到子场景 | 旅店→旅店内部、城门→城外 |
| **事件触发** | 感叹号/异常标记 | 进入叙事模式 | 夜晚的敲门声、突然出现的人影 |
| **隐藏热点** | 不可见（通幽后显示） | 特殊信息/隐藏事件 | 鬼魂、怨气痕迹、灵气异常 |
| **商店** | 商铺图标 | 打开交易界面 | 杂货铺、符纸店、丹药摊 |

### 2.3 热点的动态变化

热点不是固定不变的，会根据以下条件变化：

| 变化条件 | 效果 | 示例 |
|----------|------|------|
| **时间（白天/夜晚）** | 不同时段出现不同热点 | 夜晚：鬼市出现、茶馆关门、乞丐（鬼）蹲门口 |
| **剧情进度** | 完成某任务后新增/消失热点 | 接了悬赏后，城外出现"蛇妖出没点" |
| **线索获取** | 获得线索后解锁新热点 | 从猎户口中得知"山洞位置"→荒山出现新热点 |
| **通幽感知** | 使用通幽能力后显示隐藏热点 | 通幽后看到旅店角落有游魂、墙上有血字 |
| **多次访问** | 同一地点多次去，NPC对话变化 | 第一次：闲聊；第三次：透露秘密 |
| **名声变化** | 名声影响NPC态度和可用热点 | 百姓名声高→村民主动提供线索热点 |

### 2.4 通幽感知系统（探索核心玩法）

通幽是探索模式的**核心差异化玩法**——让"点击热点"不只是无脑点完：

```
【正常视角】
  场景背景正常显示
  只能看到普通热点（NPC、商店、入口等）

【激活通幽】（点击"通幽"按钮 或 自动触发）
  画面叠加一层"灵视"效果（色调偏青/紫，边缘模糊）
  隐藏热点显示出来（鬼魂、怨气、灵气痕迹）
  部分普通热点显示"异常"标记（NPC身上有妖气等）
  消耗法力（不能一直开着）
```

**通幽的游戏性**：
- 玩家需要**判断何时使用**——法力有限，不能到处乱开
- 某些线索**只有通幽才能发现**——推动调查
- 通幽看到的东西可能**很恐怖**——增加氛围
- 通幽后与鬼对话——获取活人不知道的信息

## 三、叙事模式详细设计

### 3.1 界面结构

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│              【全屏场景背景CG】                           │
│                                                         │
│                                                         │
│         [立绘]                          [立绘]          │
│                                                         │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  「旁白文字 / 对话文字」                                 │
│                                                         │
│                              [继续 ▶]                   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 叙事表现手段

| 手段 | 用途 | 技术实现 |
|------|------|---------|
| **旁白文字** | 环境描写、内心活动、恐怖递进 | 逐字显示，可调速 |
| **对话框+立绘** | 角色对话 | 立绘切换表情/姿态 |
| **立绘渐变** | 恐怖递进（正常→腐烂） | 立绘Alpha渐变叠加 |
| **背景切换** | 场景转换 | 淡入淡出/硬切 |
| **全屏CG** | 关键场景定格 | 全屏显示，点击继续 |
| **屏幕效果** | 恐怖/震撼 | 抖动、闪白、变色、模糊 |
| **音效** | 氛围营造 | BGM变化、环境音、突发音效 |
| **选择分支** | 玩家决策 | 2-4个选项按钮 |

### 3.3 恐怖递进的表现模板

```
第1层：日常（正常背景+正常立绘+正常BGM）
  ↓ 旁白文字暗示"微异常"
第2层：不安（BGM降调/变慢，背景微微变暗）
  ↓ 短句断行，逐字显示变慢
第3层：孤立（背景进一步变暗，立绘表情变化）
  ↓ 音效介入（远处的声音、不该有的声音）
第4层：恐怖降临（立绘渐变/背景剧变/屏幕效果）
  ↓ 极短句（1-6字），配合屏幕抖动
第5层：全貌（全屏CG或背景完全切换，真相呈现）
```

## 四、大地图设计

### 4.1 大地图界面

```
┌─────────────────────────────────────────────────────────┐
│                    【大地图】                             │
│                                                         │
│     [越州城] ●────────● [城外荒山]                       │
│         │                    │                          │
│         │              ● [蛇头山]                        │
│         │                                               │
│     [开局村落] ●────● [玉衡山]                           │
│                                                         │
│                  ● [千佛寺]（锁定/未发现）                │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  当前位置：越州城    时间：黄昏    天气：阴               │
│  [前往]  [休息至夜晚]  [查看日志]                        │
└─────────────────────────────────────────────────────────┘
```

### 4.2 地点状态

| 状态 | 表现 | 说明 |
|------|------|------|
| **未发现** | 不显示 | 玩家还不知道这个地点存在 |
| **已发现（锁定）** | 灰色图标+锁 | 知道存在但条件不满足（如需要线索/实力） |
| **可前往** | 正常图标 | 可以点击前往 |
| **有事件** | 图标上有感叹号 | 该地点有新的事件/NPC/变化 |
| **危险** | 红色边框 | 该地点当前有高危妖物出没 |

### 4.3 路途事件

从A地点前往B地点时，可能触发路途事件：

```
[越州城] ──→ 前往 ──→ [城外荒山]
                │
                ↓ （概率触发）
         【路途事件】
         "山路上遇到一个猎户，热情地邀你去他家歇脚。"
         [跟他走]  [婉拒继续赶路]  [通幽观察]
```

路途事件类型：
- 随机遭遇（山贼、散妖、流民）
- 线索事件（遇到逃难的人，透露某地信息）
- 商人（路边摊贩，可交易）
- 陷阱（食人者伪装的热情村民）

## 八、设计原则总结

| 原则 | 说明 |
|------|------|
| **每次点击有意义** | 不做"走路找东西"，每个热点都提供信息/推进/奖励 |
| **通幽是核心差异** | 通幽让同一个场景有"两层"——普通层和灵视层 |
| **时间创造变化** | 白天/夜晚让同一地点有不同内容，增加重复探索价值 |
| **叙事无缝切入** | 从探索到叙事是画面渐变，不是弹窗打断 |
| **恐怖靠节奏** | 短句+渐变+音效+屏幕效果，不靠jump scare |
| **选择有后果** | 对话选项和处置选择影响后续（名声、NPC态度、事件触发） |
| **信息渐进** | 不一次性给完所有热点，随剧情/条件逐步解锁 |
