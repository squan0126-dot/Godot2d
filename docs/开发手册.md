## 1. 环境搭建

### 1.1 安装 Godot

- 版本：**Godot 4.6.3 stable**（不要用 4.5 或 4.7 dev，API 有差异）
- 下载地址：https://godotengine.org/download
- 选 `Windows Desktop (64-bit)`，下载 `.exe` 单文件，**不需要安装**，直接双击运行

### 1.2 打开项目

方式一（推荐）：
```
双击 Godot.exe → 点"导入" → 选 C:\Users\v_qxyequan\Documents\Disha\project.godot
```

方式二（命令行）：
```
Godot_v4.6.3.exe --path "C:\Users\v_qxyequan\Documents\Disha" -e
```

### 1.3 启用 Dialogic 插件

首次打开项目必须做这一步，否则所有对话场景报错：

```
菜单栏 → 项目(Project) → 项目设置(Project Settings)
→ 插件(Plugins) 标签页
→ 找到 "dialogic"，勾选 Enable
```

> ⚠️ 如果看不到 Plugins 标签，先关闭再重新打开项目设置。

### 1.4 运行游戏

- **F5**：运行整个项目（从 Main.tscn 开始）
- **F6**：运行当前打开的场景（单独测试某个界面时用）
- **F8**：停止运行

## 3. UE → Godot 概念对照

> 你已经熟悉 UE，下面是最关键的心智模型差异。

| UE 概念 | Godot 对应 | 说明 |
|---|---|---|
| Actor | **Node** | Godot 万物皆 Node |
| Component | **子 Node** | 没有 Component，用嵌套子节点代替 |
| Blueprint (.uasset) | **Scene (.tscn) + Script (.gd)** | 一个 .tscn 就是"预制体+逻辑"的组合 |
| Level (.umap) | 也是 .tscn | Godot 的 Scene 既是关卡也是预制体 |
| GameInstance | **Autoload 单例** | 在 project.godot 里注册，全局常驻 |
| Event Dispatcher | **signal** | 一等公民，`signal MyEvent(Param: int)` |
| BeginPlay() | **_ready()** | 节点进入场景树后调用 |
| Tick() | **_process(delta)** | 每帧调用 |
| UMG | **Control 节点树** | UI 就是节点，没有第二套系统 |
| UPROPERTY(EditAnywhere) | **@export** | `@export var Speed: float = 200.0` |
| Cast To | **as 关键字** | `var p := node as Player` |
| World Outliner | **场景树面板**（左侧） | 运行时按底部 "Remote" 查看实时树 |

### 最重要的一句话

> **UE 是"类继承 + 组件挂载"，Godot 是"节点组合 + 信号通信"。**

## 4. 架构总览

### 4.1 场景切换流程

```
任意子场景
    │
    │ EventBus.RequestSceneChange.emit("res://scenes/xxx.tscn")
    ▼
EventBus（Autoload，中转站）
    │
    │ 信号传递
    ▼
Main.gd → ChangeScene()
    │  1. 销毁旧场景 (queue_free)
    │  2. 加载新场景 (load + instantiate)
    │  3. 挂到 SceneRoot 下
    ▼
新场景显示
```

**关键原则**：子场景之间**绝不直接互相引用**，所有跳转都通过 `EventBus` 发信号。

### 4.2 数据流

```
玩家操作
    │
    ▼
子场景脚本（如 DialogueScene.gd）
    │  修改数据
    ▼
GameState（Autoload）← 持有所有运行时状态
    │  序列化
    ▼
SaveSystem（Autoload）← 写入 JSON 文件
    │
    ▼
user://saves/save_0.json（磁盘）
```

### 4.3 节点树结构（运行时）

```
Main (Node)                    ← Main.gd 挂载点
├── SceneRoot (Control)        ← 当前子场景挂在这里（每次只有一个）
│   └── [当前场景，如 TitleScreen]
└── ToastLayer (CanvasLayer)   ← 全局 Toast 提示（始终在最上层）
    └── ToastLabel (Label)
```

## 5. 五大单例 API 手册

> 这 5 个单例是 Autoload，在任何脚本里**直接用名字调用**，不需要 import。

---

### 5.1 EventBus — 事件总线

**职责**：集中声明所有跨模块信号，是唯一的跨场景通信渠道。

#### 所有信号

| 信号 | 参数 | 触发时机 |
|---|---|---|
| `DialogueStarted` | `DialogueId: String` | 对话开始播放 |
| `DialogueFinished` | `DialogueId: String` | 对话结束（含跳过） |
| `DialogueChoiceMade` | `DialogueId: String, ChoiceIndex: int` | 玩家做出选项 |
| `CGShown` | `CGId: String` | CG 显示 |
| `CGHidden` | `CGId: String` | CG 隐藏 |
| `StoryNodeUnlocked` | `NodeId: String` | 故事节点解锁 |
| `FlagChanged` | `FlagName: String, Value: Variant` | 标志位变更 |
| `AchievementUnlocked` | `AchievementId: String` | 成就解锁 |
| `SaveCompleted` | `Slot: int` | 存档完成 |
| `LoadCompleted` | `Slot: int` | 读档完成 |
| `RequestSceneChange` | `ScenePath: String` | 请求切换场景 |
| `RequestToast` | `Text: String, Duration: float` | 请求显示 Toast |

#### 用法示例

```gdscript
# 发出信号（emit）
EventBus.RequestSceneChange.emit("res://scenes/title/TitleScreen.tscn")
EventBus.RequestToast.emit("已保存", 1.5)

# 监听信号（connect）
EventBus.DialogueFinished.connect(OnDialogueFinished)

# 取消监听（在 _exit_tree 里做，防止内存泄漏）
func _exit_tree() -> void:
    EventBus.DialogueFinished.disconnect(OnDialogueFinished)
```

---

### 5.2 GameState — 游戏状态

**职责**：持有当前一局的所有运行时状态，是存档的数据来源。

#### 主要字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `Variables` | `Dictionary` | 剧情变量（好感度、声望等） |
| `Flags` | `Dictionary` | 标志位（bool/int/String） |
| `UnlockedStoryNodes` | `Array[String]` | 已解锁的故事节点 ID |
| `PlayTimeSeconds` | `float` | 累计游玩时长（秒） |

#### API

```gdscript
# 标志位操作
GameState.SetFlag("met_old_taoist", true)
GameState.GetFlag("met_old_taoist")          # 返回 true / null
GameState.HasFlag("met_old_taoist")          # 返回 bool

# 故事节点
GameState.UnlockStoryNode("ch1_meet")
GameState.IsStoryNodeUnlocked("ch1_meet")    # 返回 bool

# 重置（新游戏时调用）
GameState.ResetToNewGame()

# 好感度示例（用 Variables 存）
GameState.Variables["affection_a"] = 80
var Aff: int = GameState.Variables.get("affection_a", 0)
```

---

### 5.3 SaveSystem — 存档系统

**职责**：把 GameState + AchievementManager 的数据序列化为 JSON 文件。

#### 槽位约定

| 槽位 | 用途 |
|---|---|
| slot 0 | 自动存档 |
| slot 1~9 | 手动存档 |

#### 存档文件位置

```
Windows: C:\Users\<用户名>\AppData\Roaming\Godot\app_userdata\Disha\saves\save_0.json
```

#### API

```gdscript
# 存档
SaveSystem.SaveToSlot(0, "自动存档")   # slot, 显示名
SaveSystem.SaveToSlot(1, "第一章结束")

# 读档
SaveSystem.LoadFromSlot(0)             # 返回 bool，成功/失败

# 查询
SaveSystem.HasSave(0)                  # 返回 bool
SaveSystem.GetSlotMeta(0)             # 返回 { slot, label, timestamp, play_time }
SaveSystem.ListAllSlots()             # 返回所有有存档的槽位元信息数组

# 删除
SaveSystem.DeleteSlot(1)
```

---

### 5.4 AchievementManager — 成就系统

**职责**：加载成就定义 JSON，管理解锁状态，解锁时自动弹 Toast。

#### 成就定义文件

路径：`data/achievements/achievements.json`

```json
[
  {
    "id": "first_dialogue",
    "name": "初次对话",
    "description": "完成第一段对话",
    "icon": "res://resources/cgs/ach_default.png",
    "hidden": false
  }
]
```

**新增成就只需要在这个 JSON 里加一条**，不需要改代码。

#### API

```gdscript
# 解锁成就（自动弹 Toast "🏆 解锁成就：xxx"）
AchievementManager.Unlock("first_dialogue")

# 查询
AchievementManager.IsUnlocked("first_dialogue")   # 返回 bool
AchievementManager.GetUnlockedCount()              # 已解锁数量
AchievementManager.GetTotalCount()                 # 总成就数量
AchievementManager.GetDefinition("first_dialogue") # 返回定义字典
```

### 5.5 AudioManager — 音频管理

**职责**：BGM 淡入淡出、SFX 一次性播放、主音量控制。

#### API

```gdscript
# BGM（带淡入，默认 0.5 秒）
AudioManager.PlayBGM("res://resources/audio/bgm/title.ogg")
AudioManager.PlayBGM("res://resources/audio/bgm/battle.ogg", 1.0)  # 1秒淡入
AudioManager.StopBGM()          # 默认 0.5 秒淡出
AudioManager.StopBGM(0.0)       # 立即停止

# SFX（一次性播放，不循环）
AudioManager.PlaySFX("res://resources/audio/sfx/click.ogg")

# 音量（0.0 = 静音，1.0 = 最大）
AudioManager.SetMasterVolume(0.8)
```

#### 添加音频文件

1. 把 `.ogg` 或 `.mp3` 文件放到 `resources/audio/bgm/` 或 `resources/audio/sfx/`
2. Godot 编辑器会自动 import，生成 `.import` 元数据文件
3. 直接用 `res://resources/audio/bgm/你的文件名.ogg` 路径调用

**这是游戏的核心场景**，由 Dialogic 2 驱动。

**修改当前播放的时间线**：

```gdscript
# 在 DialogueScene.gd 顶部修改这个常量
const TEST_TIMELINE := "test"   # 改成你的时间线文件名（不含 .dtl）
```

脚本：[DialogueScene.gd](../scripts/ui/DialogueScene.gd)

### 6.4 TimelineView.tscn — 故事时间轴

显示 `data/timeline/story_nodes.json` 里定义的所有故事节点。

**添加新故事节点**（只改 JSON，不改代码）：

```json
// data/timeline/story_nodes.json
[
  {
    "id": "ch1_meet",
    "chapter": 1,
    "order": 1,
    "title": "初遇",
    "description": "在画舫上第一次见到她"
  },
  {
    "id": "ch1_choice",
    "chapter": 1,
    "order": 2,
    "title": "抉择",
    "description": "葡萄事件"
  }
]
```

- `id`：唯一标识，和 `GameState.UnlockStoryNode("ch1_meet")` 里的参数对应
- `chapter`：章节号（排序用）
- `order`：章内顺序（排序用）
- 未解锁的节点显示为 `???`，解锁后显示真实标题

脚本：[TimelineView.gd](../scripts/ui/TimelineView.gd)

## 7. Dialogic 2 使用指南

### 7.1 打开 Dialogic 编辑器

```
Godot 编辑器底部面板 → Dialogic 标签页
```

或者：

```
菜单栏 → Dialogic → Open Editor
```

### 7.2 .dtl 文件语法速查

时间线文件放在 `data/timelines/` 目录下，扩展名 `.dtl`。

#### 基础对话

```
角色名: 说的话
```

```
旁白: 夜风裹着脂粉香拂过面颊。
李长安: 这里是哪里？
```

#### 旁白（无角色名）

```
水声。
月色如霜。
```

#### 分支选项

```
- 选项A的文字
    选了A之后执行的内容
    角色名: A分支的对话
- 选项B的文字
    选了B之后执行的内容
    角色名: B分支的对话
```

> ⚠️ 选项内容必须用 **Tab 缩进**，不能用空格。

#### 变量操作（好感度）

```
set affection_a += 10
set affection_a -= 5
set flag_met_a = true
```

#### 条件判断

```
if affection_a >= 80:
    角色名: 你终于来了……
elif affection_a >= 40:
    角色名: 你是谁？
else:
    角色名: 滚。
```

#### 跳转到另一条时间线

```
jump chapter_02_morning
```

#### 等待

```
wait 1.5
```

#### 触发 GDScript 信号（自定义事件）

```
signal my_custom_event
```

### 7.3 配置角色

1. Dialogic 编辑器左侧 → **Characters（角色）**
2. **+ New Character**
3. 填写：
   - **Name**：`旁白`（必须和 `.dtl` 里的名字完全一致）
   - **Display Name**：对话框里显示的名字
   - **Color**：名字颜色
4. 添加立绘（Portraits）：
   - **+ Add Portrait** → 选图片 → 起名（如 `normal`、`happy`、`sad`）

在 `.dtl` 里控制立绘：

```
join 李长安 left normal
李长安: 这里是哪里？
update 李长安 happy
李长安: 哦，原来如此。
leave 李长安
```

### 7.4 配置变量（好感度系统）

1. Dialogic 编辑器左侧 → **Variables（变量）**
2. **+ Add Variable**，添加：
   - `affection_a`，默认值 `0`
   - `affection_b`，默认值 `0`
   - `flag_met_a`，默认值 `false`

### 7.5 在 GDScript 里读取 Dialogic 变量

```gdscript
# 读取变量
var Aff: int = Dialogic.VAR.affection_a

# 设置变量
Dialogic.VAR.affection_a = 80

# 时间线结束时同步到 GameState（在 DialogueScene.gd 的 OnTimelineEnded 里）
func OnTimelineEnded() -> void:
    GameState.Variables["affection_a"] = Dialogic.VAR.affection_a
    # ... 其他逻辑
```

### 7.6 新增一条时间线（完整流程）

1. 在 `data/timelines/` 目录下新建文件，如 `chapter_01.dtl`
2. 用 Dialogic 编辑器打开并编辑，或直接用文本编辑器写
3. 在 `DialogueScene.gd` 里修改时间线名：
   ```gdscript
   const TEST_TIMELINE := "chapter_01"
   ```
4. F5 运行测试

### 7.7 多结局示例

```
# 在某条时间线末尾
if affection_a >= 80:
    jump ending_a_good
elif affection_a >= 40:
    jump ending_a_normal
else:
    jump ending_a_bad
```

对应创建三个文件：
- `data/timelines/ending_a_good.dtl`
- `data/timelines/ending_a_normal.dtl`
- `data/timelines/ending_a_bad.dtl`

## 8. 日常开发流程

### 8.1 新增一段对话

```
1. 在 data/timelines/ 新建 xxx.dtl
2. 写对话内容（参考第 7 章语法）
3. 修改 DialogueScene.gd 的 TEST_TIMELINE 常量
4. F5 运行测试
5. 测试通过后在 data/timeline/story_nodes.json 添加对应节点
6. 在对话结束回调里调用 GameState.UnlockStoryNode("节点id")
```

### 8.2 新增一个成就

```
1. 打开 data/achievements/achievements.json
2. 在数组末尾添加一条：
   {
     "id": "your_achievement_id",
     "name": "成就名称",
     "description": "解锁条件描述",
     "hidden": false
   }
3. 在合适的时机调用 AchievementManager.Unlock("your_achievement_id")
4. 不需要改任何代码
```

### 8.3 新增一个场景

```
1. 在 scenes/ 下新建文件夹和 .tscn 文件
2. 在 scripts/ui/ 下新建对应的 .gd 脚本
3. 脚本模板：

extends Control

@onready var BtnBack: Button = $BtnBack

func _ready() -> void:
    BeginPlay()

func BeginPlay() -> void:
    BtnBack.pressed.connect(OnBack)

func OnBack() -> void:
    EventBus.RequestSceneChange.emit("res://scenes/title/TitleScreen.tscn")

4. 在需要跳转到这个场景的地方：
   EventBus.RequestSceneChange.emit("res://scenes/你的目录/你的场景.tscn")
```

### 8.4 修改 UI 布局

1. 在编辑器里双击 `.tscn` 文件打开场景
2. 在左侧场景树里选中节点
3. 右侧检查器（Inspector）里修改属性
4. **Ctrl+S** 保存
5. 不需要重启，F5 直接运行看效果

### 8.5 添加 CG 图片

1. 把图片（推荐 1920×1080 PNG）放到 `resources/cgs/`
2. Godot 自动 import
3. 在脚本里使用：
   ```gdscript
   $CGSprite.texture = load("res://resources/cgs/你的图片.png")
   ```

## 9. 命名规范

本项目采用 **UE 风格命名**，方便从 UE 迁移过来的开发者阅读。

| 类型 | 规范 | 示例 |
|---|---|---|
| 变量 | 大驼峰 PascalCase | `CurrentScene`, `PlayTimeSeconds` |
| 函数 | 大驼峰 PascalCase | `BeginPlay()`, `ChangeScene()` |
| 按钮节点变量 | Btn 前缀 + 大驼峰 | `BtnNew`, `BtnSave` |
| 回调函数 | On 前缀 + 大驼峰 | `OnNewGame()`, `OnTimelineEnded()` |
| 信号名 | 大驼峰 | `RequestSceneChange`, `DialogueFinished` |
| 常量 | 全大写下划线 | `SAVE_DIR`, `MAX_SLOTS` |
| 引擎回调 | 保留下划线（不能改） | `_ready()`, `_process()`, `_exit_tree()` |
| Dialogic API | 保留 snake_case（不能改） | `timeline_ended`, `start()` |
| JSON 字段 | snake_case（兼容存档） | `"play_time_seconds"`, `"unlocked"` |

> **为什么这样设计**：GDScript 社区惯例是 snake_case，但本项目选择 PascalCase 是为了降低 UE 用户的学习成本。两种风格都是合法的 GDScript，不影响运行。

## 10. 常见报错与解决

### `Parse Error: Expected indented block after function declaration`

**原因**：Tab 和空格缩进混用，或函数体为空。

**解决**：
- 确保所有缩进用 **Tab**（不是空格）
- 如果函数暂时没有内容，加一行 `pass`：
  ```gdscript
  func MyEmptyFunc() -> void:
      pass
  ```

---

### `Parse Error: Identifier "XxxFunc" not declared in the current scope`

**原因**：函数定义在调用之后，或者函数名拼写错误。

**解决**：GDScript 不要求函数先声明后使用，通常是**拼写错误**。检查大小写是否一致。

---

### `ERROR: res://scenes/xxx.tscn - Parse Error: ...`

**原因**：场景文件引用的脚本有语法错误。

**解决**：先修复脚本错误，再运行场景。

---

### 按钮点击没反应

**原因**：通常是 `mouse_filter` 设置问题，父节点拦截了鼠标事件。

**解决**：
- 检查按钮的父节点（Control 类型）的 `Mouse Filter` 属性
- 容器节点设为 `Ignore`，按钮本身设为 `Stop`

---

### Dialogic 时间线不存在

**原因**：`.dtl` 文件名和 `Dialogic.start()` 里的参数不匹配。

**解决**：
- 文件名：`data/timelines/chapter_01.dtl`
- 调用：`Dialogic.start("chapter_01")`（不含路径，不含扩展名）

---

### 存档读不到

**原因**：存档文件路径在 `user://` 目录，不在项目目录里。

**查找存档文件**：
```
Windows: C:\Users\<用户名>\AppData\Roaming\Godot\app_userdata\Disha\saves\
```

---

### 插件报错 / Dialogic 功能不可用

**原因**：插件未启用。

**解决**：
```
项目设置 → 插件 → dialogic → 勾选 Enable
```
